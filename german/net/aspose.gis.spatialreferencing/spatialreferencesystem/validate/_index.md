## SpatialReferenceSystem.Validate method

Stellen Sie fest, ob diese SRS gültig ist.

```csharp
public abstract bool Validate(out string errorMessage)
```

| Parameter | Typ | Beschreibung |
| --- | --- | --- |
| errorMessage | String& | Wenn die Methode zurückkehrt`false` dann ist dies eine Beschreibung der Ungültigkeit. |

### Rückgabewert

`true` wenn SRS gültig ist,`false` Andernfalls.

### Bemerkungen

Ein gültiges SRS muss ein gültiges Ellipsoid haben. - Das geografische SRS muss genau eine Ost/West-Achse, genau eine Nord/Süd-Achse und eine optionale Aufwärts-/Abwärtsachse haben (optionale Achse ist vorhanden, wenn das geografische SRS eine Verbindung aus dem geografischen 2D-SRS und vertikales SRS). - Das projizierte SRS muss genau eine Ost/West-Achse, genau eine Nord/Süd-Achse und eine optionale Aufwärts-/Abwärtsachse haben (optionale Achse ist vorhanden, wenn das projizierte SRS eine Verbindung aus dem geografischen 2D-SRS und dem vertikalen SRS ist). - Geozentrische SRS muss genau eine Ost/West-Achse, genau eine Nord/Süd-Achse und genau eine andere Achse haben. - Vertikale SRS muss genau eine Auf-/Ab-Achse haben. - Lokale SRS muss mindestens eine Achse haben. Achsenrichtungen werden nicht gemessen. - Verbund-SRS muss eine Kombination aus einem gültigen geografischen/projizierten und einem gültigen vertikalen SRS sein.
