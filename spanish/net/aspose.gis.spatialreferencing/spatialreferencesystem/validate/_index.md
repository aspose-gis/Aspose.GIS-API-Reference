## SpatialReferenceSystem.Validate method

Determinar si este SRS es válido.

```csharp
public abstract bool Validate(out string errorMessage)
```

| Parámetro | Escribe | Descripción |
| --- | --- | --- |
| errorMessage | String& | Si el método devuelve`false` entonces esta es la descripción de la invalidez. |

### Valor_devuelto

`true` si SRS es válido,`false` de lo contrario.

### Observaciones

El SRS válido debe tener un elipsoide válido. - El SRS geográfico debe tener exactamente un eje este/oeste, exactamente un eje norte/sur y un eje arriba/abajo opcional (el eje opcional está presente cuando el SRS geográfico es un compuesto de SRS geográfico 2D y SRS vertical). : el SRS proyectado debe tener exactamente un eje Este/Oeste, exactamente un eje Norte/Sur y un eje Arriba/Abajo opcional (el eje opcional está presente cuando el SRS proyectado es un compuesto de SRS geográfico 2D y SRS vertical). - El SRS geocéntrico debe tener exactamente un eje Este/Oeste, exactamente un eje Norte/Sur y exactamente un eje Otro. - El SRS vertical debe tener exactamente un eje Arriba/Abajo. - El SRS local debe tener al menos un eje. Las direcciones de los ejes no miden. : el SRS compuesto debe ser una combinación de un SRS geográfico/proyectado válido y un SRS vertical válido.
